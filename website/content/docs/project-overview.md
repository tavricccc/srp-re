# 專案總覽

[English](en/project-overview.md) · [文件首頁](README.md)

Novae（Novae, Novae）是一套供校內社群提出、討論、附議與追蹤議題的開源 Web App。它把公開議題、私密權益案件、設備回報、校方公告與後續通知放進一套可設定的流程，同時將作者身分與敏感資料留在受控邊界內。

## 為什麼需要這個專案

表單或社群貼文容易建立，卻難以持續追蹤。Novae 補足分類規則、審核、狀態、回覆期限、附議進度、通知、備份與維運視角，讓管理單位與學生看到同一份可追蹤紀錄。

## 核心能力

| 領域 | 能力 |
| --- | --- |
| 身分 | 限定校內 Google 帳號登入，管理員角色由後端同步 |
| 提案 | 分類、搜尋、留言、分享、審核、狀態與期限追蹤 |
| 隱私 | 每個分類可設定公開、審核後校內可見、或僅作者與管理員可見 |
| 附議 | 可按分類啟用門檻與期限，支援取消與即時進度 |
| 公告 | 管理員發布、編輯與刪除；使用者可按讚與留言 |
| 通知 | 站內通知、Web Push、個人與主題式推播 |
| 媒體 | 瀏覽器壓縮 WebP、簽名上傳、短效簽名讀取網址 |
| 維運 | 管理儀表板、背景工作、資料保留、失敗追蹤與自動部署 |

## 適用情境

- 學生會、系學會或自治組織需要公開提案與附議。
- 校內單位需要同時處理公開議題與私密權益案件。
- 小型團隊想以託管服務快速建立可維護的平台。
- 開發者想參考 Vue、Supabase 與 Firebase Auth 的完整整合。

## 不適用情境

- 禁止使用外部雲端服務或要求完全離線、自架的環境。
- 需要電子簽章、法定申訴、公文交換或不可否認性稽核的流程。
- 需要多租戶計費、跨校組織隔離或企業級身分治理的 SaaS。
- 無人能持續管理第三方服務、金鑰、帳單、備份與安全事件。

## 專案邊界

Novae 是可自行部署的應用程式，不是代管服務。採用者必須自行建立 Firebase、Supabase、Vercel、Cloudinary、Notion 與 Upstash 資源，並負責當地法規、告知與同意、資料保留、管理員任命、帳務及事故處理。

目前介面以單一學校、單一允許網域及單一管理員群組為設計前提。Firebase 僅負責登入、App Check 與推播；主要資料與領域流程位於 Supabase，圖片位於 Cloudinary，Notion 是外部副本而不是主要資料庫。

## 成熟度與非目標

Repository 具備型別檢查、lint、build、Edge Function 型別檢查、架構回歸測試與分離的前後端部署工作流。這些保護降低部署風險，但不等同服務等級協議、安全認證或法規認證。專案目前也不承諾向後相容的公開 API。
