# 使用手冊

[English](en/user-guide.md) · [文件首頁](README.md)

本手冊提供平台使用者與管理員的操作流程。實際分類名稱、可見範圍、附議門檻與期限由部署者設定，因此畫面可能與範例不同。

## 角色與可見範圍

| 角色 | 可以做什麼 |
| --- | --- |
| 訪客 | 前往登入頁；不能讀取校內內容 |
| 校內使用者 | 瀏覽有權限的提案與公告、新增提案、留言、附議、管理自己的內容與通知 |
| 提案作者 | 讀取自己的審核中或私密提案，追蹤狀態與管理自己的提案 |
| 管理員 | 審核提案、調整狀態、發布公告、處理內容、查看管理 Dashboard |

畫面是否隱藏按鈕只是使用體驗；真正權限由後端規則決定。看到內容不代表能編輯或管理它。

## 登入與安裝

1. 選擇「使用 Google 登入」。
2. 使用已驗證且符合學校網域的帳號。
3. 第一次登入時，平台會同步使用者資料與角色。
4. 支援的瀏覽器可將平台安裝成 PWA；推播通知仍需另行授權。

若帳號正確但無法登入，先確認 email 網域與驗證狀態，再聯絡管理員。管理員權限更新後可能需要重新登入才能取得新 token。

## 尋找提案

提案看板可依分類與狀態瀏覽，並使用搜尋尋找標題或內容。列表只會顯示目前帳號有權閱讀的資料：

- `school` 分類：所有已登入校內使用者可見。
- `reviewed-school` 分類：審核前只有作者與管理員可見；通過後校內可見。
- `owner-admin` 分類：始終只有作者與管理員可見。

「我的提案」包含目前使用者建立且仍有權讀取的內容。分享連結不會繞過登入與權限，收件者仍可能看到無權存取提示。

## 建立提案

1. 開啟新增選單並選擇提案。
2. 選擇最符合內容的分類；先閱讀分類帶來的可見性與作者顯示結果。
3. 填寫標題與 Markdown 內容，必要時插入圖片。
4. 預覽內容，確認沒有不應公開的姓名、聯絡方式或證件資訊。
5. 送出後依分類進入公開、待審核或私密流程。

每個分類的行為：

| 設定 | 送出後的使用者體驗 |
| --- | --- |
| `readAccess: school` | 送出後校內使用者可依狀態與列表規則瀏覽 |
| `readAccess: reviewed-school` | 顯示「待審核」；核准後公開並在啟用時開放附議，未通過時只有作者與管理員看到原因 |
| `readAccess: owner-admin` | 不進入全校公開列表，作者可持續從自己的提案查看 |
| `authorVisible: true` | 有權閱讀者可看到作者資訊 |
| `authorVisible: false` | 一般內容畫面以匿名方式呈現，但管理端仍保存作者關聯 |

圖片上傳前會在裝置中壓縮。若超過檔案大小、像素或張數限制，平台會拒絕或降低品質；不要重複上傳同一張圖來繞過限制。

## 附議與留言

只有 `support.enabled: true` 的分類會顯示附議。使用者可附議或取消附議，進度以 `goal` 為目標，操作只在 `deadlineDays` 期限內有效。達標代表觸發平台設定的處理流程，不等於提案必然採納。

留言支援 Markdown 與有限圖片。留言可形成回覆討論；通知會依事件與收件者設定送出。請勿在公開討論貼出私密案件內容或他人個資。

## 提案狀態

| 狀態 | 意義 |
| --- | --- |
| 待審核 | 需要管理員判斷是否公開 |
| 審核未過 | 未公開；作者可查看管理員填寫的原因 |
| 處理中 | 管理單位已開始處理，可追蹤回覆期限 |
| 已完成 | 已結案且應附上結果說明 |
| 不可行 | 已結案並說明無法執行的原因 |

狀態名稱可能只在適用的分類或管理流程出現。結果說明是對使用者的正式回覆，應避免內部術語與未經確認的承諾。

## 公告

所有有權限的校內使用者可閱讀公告、按讚與留言。管理員可新增、編輯或刪除公告，並用 Markdown 和圖片整理內容。公告可能透過站內通知與 Web Push 廣播；發布前請確認標題、連結、日期與對象。

## 通知

通知頁保存站內通知並顯示未讀狀態。點選通知時，平台會再次確認目標內容的權限；已刪除或無權閱讀的內容不會因通知連結而公開。

Web Push 需要瀏覽器支援、系統允許、網站權限與有效裝置 token。拒絕權限後可從瀏覽器網站設定重新開啟。關閉 Push 不影響站內通知。

## 管理員流程

### 審核提案

1. 從待審核清單開啟提案。
2. 檢查分類是否正確、內容是否適合公開、是否包含個資或不安全媒體。
3. 選擇審核通過，或填寫具體且可行動的未通過原因。
4. 核准後確認公開範圍、作者顯示與附議設定符合預期。

### 更新處理狀態

將已受理提案標記為處理中；結案時選擇已完成或不可行並填寫結果。不要用狀態取代與作者必要的私密溝通。

### Dashboard 與日常管理

Dashboard 用於查看平台統計、分類成果、近期活動與背景工作異常。它不是稽核系統；異常時依[維運手冊](operations.md)查看 logs 與供應商狀態。

## 使用者安全建議

- 提案前先判斷分類是否公開、是否顯示作者。
- 即使選擇匿名顯示，管理員仍可基於營運需要識別作者。
- 不上傳密碼、證件、醫療資料或不必要的第三人個資。
- 不在通知截圖或公開 issue 洩漏私密提案內容。
- 發現越權或資料洩漏時停止操作並依[安全回報流程](security.md#回報安全問題)聯絡維護者。
