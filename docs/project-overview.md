# 專案總覽

[English](en/project-overview.md) · [文件首頁](README.md)

Novae（Novae, Novae）是一套供校內社群提出、討論、附議與追蹤議題的開源 Web App。它把公開議題、私密權益案件、設備回報、校方公告與後續通知放進一套可設定的流程，同時將作者身分與敏感資料留在受控邊界內。

## 為什麼需要這個專案

表單或社群貼文容易建立，卻難以持續追蹤。Novae 補足分類規則、審核、狀態、回覆期限、附議進度、通知、備份與維運視角，讓管理單位與學生看到同一份可追蹤紀錄。

## 核心能力

| 領域 | 能力 |
| --- | --- |
| 身分 | 限定校內 Google 帳號登入，管理員角色由後端同步 |
| 提案 | 分類、搜尋、留言、分享、審核、狀態與期限追蹤 |
| 隱私 | 每個分類可設定公開、審核後校內可見、或僅作者與管理員可見 |
| 附議 | 可按分類啟用門檻與期限，支援取消與即時進度 |
| 公告 | 管理員發布與刪除；發布後不可編輯，使用者可按讚與留言 |
| 通知 | 站內通知、Web Push、個人與主題式推播 |
| 媒體 | 瀏覽器壓縮 WebP、簽名上傳、短效簽名讀取網址 |
| 維運 | 管理儀表板、背景工作、資料保留、失敗追蹤與自動部署 |

## 適用情境

- 學生會、系學會或自治組織需要公開提案與附議。
- 校內單位需要同時處理公開議題與私密權益案件。
- 小型團隊想以託管服務快速建立可維護的平台。
- 開發者想參考 Vue、Supabase 與 Firebase Auth 的完整整合。

## 彈性自訂

- 依分類決定校內公開、審核後公開或僅當事人與管理員可見。
- 個別設定作者顯示、附議門檻、附議期限與回覆期限。
- 以校名、允許網域、管理員名單與通知偏好打造自己的版本。
- 學生端與管理端共用同一套狀態，從提出、審核到結案都能追蹤。

## 一套完整的校園參與體驗

Novae 提供可自行部署的完整應用程式。學生可提出、附議、留言與追蹤；管理團隊可審核、回應、發布公告並掌握近期活動。桌面以可展開側邊導覽集中主要功能與新增入口，手機使用底部導覽；內容會依實際可用寬度調整排列，兩端仍共用相同資料與進度，也能安裝到主畫面並接收通知。
